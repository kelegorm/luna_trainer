# Tango Trainer — Концептуальный документ

> Тренажёр для игры Tango (LinkedIn). Цель — развить скорость и точность распознавания логических паттернов, ведущих к форсированным ходам.

---

## Правила Tango (кратко)

- Поле 6×6
- Два символа: ☀ (солнце) и ☾ (луна)
- В каждой строке и каждом столбце ровно 3 солнца и 3 луны
- Нельзя поставить три одинаковых символа подряд (по горизонтали или вертикали)
- Между некоторыми клетками стоят знаки `=` (одинаковый символ) или `×` (разный символ)

---

## Дедакшены — форсированные логические выводы

### 1. `line_completion` — завершение линии

Если в строке или столбце заполнены 5 из 6 клеток, шестая форсирована автоматически.

```
☀ ☾ ☀ ☾ ☀ _   →   последняя = ☾
```

**Приоритет в тренировке: высокий.** Это самый простой паттерн, но мозг склонен его игнорировать в поиске сложных ходов.

---

### 2. `no_three` — запрет тройки

Три подпаттерна:

```
no_three_pair_left:   ☀ ☀ _  →  _ = ☾
no_three_pair_right:  _ ☀ ☀  →  _ = ☾
no_three_gap:         ☀ _ ☀  →  _ = ☾
```

Применяется по горизонтали и вертикали.

---

### 3. `balance` — баланс символов

Если в линии из 6 клеток уже стоит 3 символа одного типа, все оставшиеся пустые клетки — другого типа.

```
☀ _ ☀ _ ☀ _   →   все _ = ☾
```

---

### 4. `edge_trap` — ловушка на краях

**edge_1_5_trap**: если позиции 1 и 5 заняты одним символом A:

```
A _ _ _ A _   →   позиция 6 = B
```

Логика: если поставить A на 6-ю позицию, то позиции 2, 3, 4 должны все быть B — три подряд, запрещено. Значит, позиция 6 = B.

**edge_2_6_trap**: аналогично для позиций 2 и 6:

```
_ A _ _ _ A   →   позиция 1 = B
```

---

### 5. `relation` — знаки между клетками

```
relation_same:     X = Y   →   Y имеет тот же символ, что X
relation_opposite: X × Y   →   Y имеет противоположный символ

Цепочки:
  X = Y × Z = W   →   если X = ☀, то Y = ☀, Z = ☾, W = ☾
```

Цепочки могут тянуться через несколько клеток.

---

## Solving Rhythm — ритм решения

### Propagation mode (локальное распространение)

После каждого нового хода **не переключаться сразу** на поиск по всему полю. Сначала проверить локальный контекст:

1. Строка, в которую поставлен ход
2. Столбец, в который поставлен ход
3. Соседние клетки (±1)
4. Клетки, связанные знаками `=` и `×`

Каждая новая поставленная клетка — новый "очаг" для Propagation.

### Hunt mode (глобальный скан)

Когда локальная цепочка иссякла, переходить к глобальному сканированию в следующем порядке:

| Приоритет | Тип |
|-----------|-----|
| 1 | `line_completion` — линии 5/6 |
| 2 | `no_three` — пары и зазоры |
| 3 | `balance` — 3 одинаковых в линии |
| 4 | `relation` — знаки = и × с известной клеткой |
| 5 | `edge_trap` — паттерны на позициях 1-5, 2-6 |

> Порядок отражает типичную сложность распознавания, не жёсткое правило. В реальной игре мозг берёт что видит быстро.

## Режимы тренировки

### Слой 1 — Pure Pattern Drill

- Поле содержит **ровно один** целевой дедакшен
- 70% упражнений: паттерн присутствует → нужно найти и кликнуть
- 30% упражнений: паттерн отсутствует → нужно нажать «не нашёл»
- Вокруг — реалистичный шум, не учебный мусор
- Включает **negative examples**: похожие, но невалидные паттерны

**Цель:** выработать автоматическое зрительное узнавание конкретного паттерна.

---

### Слой 2 — Distracted Pattern Drill

- Целевой паттерн присутствует всегда
- Вокруг — правдоподобный шум и похожие, но невалидные паттерны
- Сложнее, чем Слой 1 за счёт визуального шума

**Цель:** находить паттерн несмотря на отвлекающий контекст.

---

### Слой 3 — Real Solve

- Полноценное поле, смешанные паттерны
- Любой валидный форсированный ход засчитывается
- Телеметрия фиксирует время нахождения каждого типа паттерна
- После решения: система показывает что было пропущено и что находилось медленно

---

## Targeted Weakness Training

Цикл:

```
Real Solve → телеметрия → выявить самый медленный паттерн
    → назначить целевой drill → Real Solve → ...
```

Система автоматически увеличивает частоту drill для слабых паттернов.

**Пример:** `edge_1_5_trap` — среднее время 4.2 с → система добавляет больше `edge_1_5` упражнений в ротацию.

---

## Штрафная система и negative examples

Без negative examples мозг начинает давать ложные срабатывания (false positives — «видеть лица в облаках»).

| Действие | Результат |
|----------|-----------|
| Правильный клик на паттерн | +0 с (норма) |
| Ложный клик (паттерна нет) | +1 с к времени |
| Пропуск существующего паттерна | +2 с к времени |
| Правильное «не нашёл» | бонус (ускорение) |

---

## Возможная аналитика (heatmap)

- Время нахождения паттерна в строках vs столбцах
- Время для ☀ vs ☾ в роли триггера
- Паттерны в центре поля vs по краям

---

## Принципы дизайна тренажёра

1. **Реалистичный шум.** Фоновые клетки должны выглядеть как настоящая игра, не как пустое поле с одним паттерном.
2. **Нет жёсткого порядка поиска.** Тренажёр принимает любой валидный форсированный ход — туннельное зрение хуже гибкого сканирования.
3. **Слабые паттерны — в приоритет.** Цель не равномерная практика, а целенаправленное устранение узких мест.
4. **Propagation важнее Hunt.** Умение дожать локальную цепочку снижает общее время решения больше, чем скорость глобального сканирования.
